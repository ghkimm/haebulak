# Handoff: 해부락개론 — 포트폴리오 컨설팅 웹 서비스 (1차)

## Overview
해부락개론은 4주 과정의 포트폴리오 컨설팅 서비스입니다. 참가자는 주차별 과제를 웹에서 작성/제출하고, 강사는 별도 화면에서 코멘트와 답변을 남깁니다.

4주 구성:
- 1주차 상담 — 지금 하는 일 / 앞으로 하고 싶은 일 / 수강 이유 작성
- 2주차 방향과 이력 — 방향 서술 + 모든 이력을 카드로 쌓기, 강사 피드백/코멘트를 함께 확인, 피드백 아래에 질문 남기기
- 3주차 축약과 구조 — (아직 구성 중)
- 4주차 제작 — (아직 구성 중)

## About the Design Files
이 번들의 파일은 **HTML로 만든 디자인 레퍼런스(프로토타입)** 입니다. 의도한 룩앤필과 동작을 보여주는 목업이며, 그대로 배포할 프로덕션 코드가 아닙니다. 목표는 이 디자인을 대상 코드베이스의 기존 환경(React/Vue/Next 등)과 패턴/라이브러리로 **재구현**하는 것입니다. 환경이 아직 없다면 프로젝트에 맞는 프레임워크를 선택해 구현하면 됩니다.

파일은 스트리밍 렌더 런타임(support.js) 위에서 동작하는 컴포넌트라, 로직/상태 구조는 참고만 하고 실제 상태관리는 대상 스택의 관례를 따르세요.

## Fidelity
**High-fidelity.** 색상·타이포·간격·상호작용이 최종안에 가깝습니다. 아래 디자인 토큰과 규칙을 지켜 픽셀에 가깝게 재현하세요. 데이터(참가자/이력/코멘트)는 데모용 더미이며 실제 API로 대체합니다.

## 디자인 원칙 (반드시 준수)
차분한 개인 작업실 톤. 학원/LMS 느낌 금지. 여백 넉넉, 한국어 존댓말.

**강조색(#13724a)은 아래 네 곳에만 사용:**
1. 서비스 이름 (현재 1차에서는 워드마크 미표기 — 이름 텍스트가 들어갈 경우 강조색)
2. 현재 진행 중인 주차 표시 (진행 표시 바/라벨)
3. 주요 버튼 (들어가기, 제출하기, 강사 저장/등록 등 primary)
4. 입력 칸 포커스 시 테두리

**강조색을 쓰지 않는 곳:** 완료 상태(체크/흐린 회색으로만 구분), 본문 글자, 안내 문구, 카드 배경.

**전체 금지사항:**
- 초록 계열 2가지 이상 사용 금지 (#13724a와 hover용 #0e5637만 허용)
- 신호등 배색(성공=초록/실패=빨강) 금지
- 점수, 등급, 진행률 %, 참가자 간 비교 금지
- "WEEK" 등 영어 표기 금지. "1주차, 2주차"로만
- 그라데이션, 배경 이미지, 그림자 남발 금지 (얕은 그림자 1개 정도만: 드롭다운 메뉴)

## Screens / Views

### 1. 진입 (파일: `01 진입.dc.html`)
- **Purpose**: 이름 + 발급받은 참가 코드로 자기 페이지에 들어감. 로그인 페이지가 아니라 "작업실 문 앞".
- **Layout**: 화면 전체 중앙 정렬(flex center). 콘텐츠 폭 max-width 380px, 세로 흐름.
- **Components**:
  - 한 줄 설명 텍스트 (선택). 현재 1차에서는 워드마크/설명 모두 제거된 상태 — 이름/코드 입력만.
  - 이름 입력: label "이름" 13px #4a4a4a 500 / input padding 13px 14px, 15px, border 1px #d8d8d8, radius 6px, focus border #13724a.
  - 참가 코드 입력: label "참가 코드" / 같은 스타일 / 아래 안내 12px #9a9a9a "개별로 전달드린 코드예요."
  - 들어가기 버튼: 폭 100%, padding 14px, 15px 600, 흰 글자, 배경 #13724a, radius 6px, hover #0e5637.
  - 하단 안내 12px #9a9a9a 중앙 "코드를 모르시면 담당자에게 연락 주세요."
- **금지**: 소셜 로그인, 회원가입, 비밀번호 찾기.

### 2. 작업실 — 참가자용 (파일: `02 작업실.dc.html`)
- **Purpose**: 참가자가 주차별 과제를 작성/제출하고 강사 피드백을 확인, 질문을 남김.
- **Layout**: max-width 720px 중앙, 상단 40px 패딩, 하단 고정 바 때문에 padding-bottom 96px.
- **상단 주차 진행 표시**: 4개 컬럼(flex, gap 6px). 각 컬럼 = 상단 3px 바 + "N주차" 라벨 + 상태 텍스트. 클릭 시 해당 주차로 전환.
  - 진행 중 주차(2주차): 바/라벨/상태 모두 #13724a, 라벨 700.
  - 완료(1주차): 바 #cfcfcf, 라벨 #8a8a8a, 상태 #b0b0b0.
  - 대기(3·4주차): 바 #ececec, 라벨 #b8b8b8, 상태 #c4c4c4.
  - 지금 보고 있는 주차: 컬럼 배경 #f0f0ef (강조색 아님 — 선택 표시는 무채색으로만).
- **1주차 뷰**: 제목 "상담 단계"(26px 700) + 부제 "정답은 없어요. 지금 생각을 편하게 적어 주세요." 세로 3개 섹션(항목1 지금 하고 있는 일 / 항목2 앞으로 하고 싶은 일 / 항목3 이 수업을 듣는 이유), 각 textarea min-height 130px.
- **2주차 뷰**:
  - 헤더: 제목 "기록 단계", 부제 "활동했던 이력을 꺼내 적고, 강사 피드백을 함께 확인하세요…", 마감 표시 13px #9a9a9a.
  - 항목1 나아가고자 하는 방향: textarea min-height 200px. 아래 강사 피드백 블록은 **코멘트가 있을 때만** 렌더(빈 상자 금지). 피드백 블록 배경 #f5f5f4, radius 8px. 그 안에 참가자가 남긴 질문 리스트("내 질문" + 본문 + "답변을 기다리는 중이에요" 12px #a8a8a8) + 질문 입력 textarea + "질문 남기기" 버튼(무채색 아웃라인).
  - 항목2 지금까지의 모든 이력: 안내 문구 + 이력 카드 목록(gap 12px) + 점선 "+ 이력 추가하기" 버튼.
    - 카드(보기 상태): 좌측 순번(700 #b0b0b0) + 단체명(15px 600) + 대주제(14px #4a4a4a) + 설명(14px #555). 우측 상단 수정/삭제 텍스트 버튼. 강사 코멘트 블록은 **코멘트 있을 때만** (배경 #f5f5f4), 그 아래 질문 리스트 + 질문 입력.
    - 카드(편집 상태): 단체명 input / 대주제 input / 설명 textarea + 삭제·완료 버튼.
    - 신규 추가 카드는 편집 상태로 시작, 강사 코멘트 없음.
- **하단 고정 바**(1·2주차에서만): position fixed, 반투명 흰 배경 + blur, 상단 1px #ececec. "마지막 저장 · {시각}" + 임시 저장(무채색 아웃라인) + 제출하기(#13724a primary).
- **3·4주차 뷰**: 중앙 정렬 "3주차 · 축약과 구조" / "4주차 · 제작" 제목 + "아직 구성 중이에요." 회색. 하단 바 숨김.
- **주의**: 글을 오래 쓰는 화면이라 입력 칸 넉넉히. 모바일 대응 필수(720px 이하에서 패딩 유지, 고정 바 전체 폭).

### 3. 강사 관리 — 강사용 (파일: `03 강사관리.dc.html`)
- **Purpose**: 참가자를 골라 방향/이력마다 코멘트를 남기고, 참가자 질문에 답변.
- **Layout**: max-width 1000px 중앙. 좌측 리스트 없음 — 상단 참가자 이름이 드롭다운 토글.
- **헤더**: "강사 관리"(22px 700) + 안내 문구.
- **참가자 선택 드롭다운**: 이름(20px 700) + "▾" 버튼. 클릭 시 아래로 메뉴(절대위치, 흰 배경, border 1px #e2e2e2, radius 10px, box-shadow 0 6px 20px rgba(0,0,0,0.08)). 항목마다 이름 + 코드(port01, port02…) + "답변 대기 질문 N개"(있을 때). 선택 시 전환하고 닫힘. 이름 옆에 "{주차} 진행 중" 13px 600 #13724a.
- **주차 탭**: 작업실과 동일한 4컬럼 전환 UI(진행 중=강조색, 보는 중=#f0f0ef).
- **1주차 뷰**: 참가자 답변 3개를 읽기 전용 박스(배경 #fafafa, border #ededed)로 표시 + "상담 메모" textarea + 메모 저장 버튼(primary).
- **2주차 뷰**:
  - 항목1 방향: 참가자 방향 읽기 전용 박스 + "강사 피드백" textarea + "피드백 저장" 버튼(primary) + 저장 시 "저장되었습니다" 회색 텍스트. 참가자 질문마다 "참가자 질문" 표시 + 답변 textarea + "답변 등록"; 등록 후 "내 답변"으로 표시.
  - 항목2 이력: 카드마다 단체명/대주제/설명 읽기 전용 + "강사 코멘트" textarea + "코멘트 저장" + 질문 답변(위와 동일).
- **데이터 흐름**: 여기서 저장한 피드백/코멘트/답변이 참가자 작업실(화면2)의 강사 피드백/코멘트/답변 자리에 나타나야 함. 현재 두 화면은 데이터 미연동 목업.

## Interactions & Behavior
- 주차 탭/드롭다운 클릭으로 뷰 전환 (라우팅 또는 로컬 상태).
- 이력 카드: 추가(편집상태로 생성) → 완료(보기 전환) / 수정(편집 전환) / 삭제.
- 질문 남기기/답변 등록: 빈 입력이면 무시(trim), 등록 시 리스트에 추가하고 입력 비움.
- 피드백/코멘트/메모 저장: 저장 후 "저장되었습니다" 표시, 편집하면 표시 해제.
- 강사 피드백·코멘트 블록과 질문/답변 블록은 **내용이 있을 때만** 렌더 (빈 상자 절대 금지).
- 임시 저장: "마지막 저장 · {시각}" 갱신.
- 며칠에 나눠 작성하므로 자동/임시 저장으로 입력 보존 권장.
- 전환/호버는 background/border-color 0.15s 정도의 잔잔한 트랜지션.

## State Management
- 진입: name, code (로그인/코드 검증 → 참가자 페이지로).
- 작업실: activeWeek(1~4), 1주차(now/want/why), 2주차(direction + directionFeedback + directionQuestions[], items[{org,topic,desc,comment,questions[],editing}]), lastSaved.
- 강사 관리: participants[], selected, activeWeek, dropdownOpen. 참가자별 w1(now/want/why + 상담메모), direction+feedback, directionQuestions[{text,answer}], items[{…,commentDraft, questions[{text,answer}]}].
- 실제 구현에서는 참가자 데이터/코멘트/질문을 API로 fetch·persist. 강사 저장이 참가자 뷰에 반영되도록 공유 소스 필요.
